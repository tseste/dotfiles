## Code Style (All Languages)
- Write clean, readable code over clever one-liners.
- Use descriptive variable and function names. Avoid single-letter names except in loops or math.
- Keep functions small and focused on a single responsibility.
- Add comments only where the intent is non-obvious — not to describe what the code does.
- Prefer explicit over implicit. Avoid magic numbers; use named constants.
- Handle errors and edge cases explicitly — don't silently swallow exceptions.
